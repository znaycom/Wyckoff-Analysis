# Wyckoff Strategy Playbook (量化架构视角)

本文档描述策略框架与量化执行流程，整体架构采用“日线级 + 结构化强风控 + 人机协同”的设计理念：
通过漏斗广域初筛、TopN 量化打分兜底、大模型语义盘口决策，以及 OMS 端的严格风控，最终输出确定下的交易建议。

覆盖模块：
- Step2 漏斗选股：`scripts/wyckoff_funnel.py` + `core/wyckoff_engine.py` (L1-L4 数学形态初筛与打分)
- Step3 批量 AI 研报：`scripts/step3_batch_report.py` (LLM 三阵营战报构建)
- Step4 私人再平衡 OMS：`scripts/step4_rebalancer.py` (持仓管理与实盘风控阻断)

## 0. 核心概念与技术术语

> **完整术语速查手册请参阅 [`GLOSSARY.md`](GLOSSARY.md)**，涵盖回测指标、Wyckoff 方法论、漏斗管线、板块轮动、大盘水温、watch_score 评分公式、OMS 风控等全部概念。
>
> 以下仅列出本文后续章节频繁引用的缩写，方便快速对照：

| 缩写 | 全称 | 一句话 |
|------|------|--------|
| RPS | Relative Price Strength | 涨幅超越全市场 N% 股票的百分位排名 |
| RS | Relative Strength | 个股涨幅 - 大盘涨幅 |
| ATR | Average True Range | 平均真实波动率，用于自适应止损 |
| SOS | Sign of Strength | 放量突破——吸筹结束的发令枪 |
| LPS | Last Point of Support | 缩量回踩——最后支撑点 |
| EVR | Effort vs Result | 量价背离——放量不跌，主力承接 |
| MA | Moving Average | 移动平均线（MA50 / MA200） |
| SLTP | Stop Loss & Take Profit | 止损止盈退出机制 |
| VaR | Value at Risk | 95% 置信下单笔最大损失 |
| CVaR | Conditional VaR | 击穿 VaR 后的平均尾部损失 |

## 1. 系统调度与数据工程

### 1.1 定时与环境对齐
- 由 GitHub Actions 在工作日自动化调度（同分支并发互斥机制避免任务重叠）。
- 从 `v2.0.0` 起，Web 端的 `Wyckoff Funnel` 与批量 `AI 分析` 也统一改为 **Streamlit 前台触发 + GitHub Actions 后台执行**。页面只负责提交参数、查看状态、读取结果 artifact，不再在 Streamlit 进程内重算全量漏斗。
- 统一切换目标交易日（T / T-1）：由 `utils/trading_clock.py` 判定，严格依据盘后及盘中时间节点切分。
- **窗口口径统一：** 漏斗、Step3、Step4、回测与导出默认回看窗口统一为 **320 个交易日**，单股大师模式同样按近 320 个交易日输入，确保 MA200 计算稳定且跨模块一致。
- **单股超时口径：** 单股分析默认 `LLM_REQUEST_TIMEOUT=90s`、`LLM_TOTAL_TIMEOUT=240s`，减少复杂输入下的误超时中断。
- **快照补齐与对齐检查：** 若三方日线数据未落库，可依靠本地 Snapshot 补齐最后一根 K 线进行计算。

### 1.2 Snapshot 快照与本地回测器
- 每轮日线全量拉取后，会将 4000+ 股票行情快照序列化到本地 `data/funnel_snapshots` 目录。
- **意义：** 从根本上分离了“网络取数”与“量化算子”，系统支持秒级重播（Replay），大幅提升了因子调优、正则参数改进以及回测的工程效率。
- **当前默认回测口径（2026-04）**：`hold_days=30`、`exit_mode=sltp`、`stop_loss=-7%`、`take_profit=18%`（盈亏比 2.57:1）；摩擦成本买卖各 0.5%；入场口径为信号日收盘后次日开盘买入（消除前视偏差）。在 summary 中强制输出 `最大回撤 / VaR95 / CVaR95 / 最长连亏` 四项风险指标。
- **偏差声明（强制）**：回测器默认关闭“当前截面市值/行业映射过滤”（可用 `--use-current-meta` 显式开启旧口径），以降低 look-ahead bias；但仍存在幸存者偏差（股票池基于当前在市样本）。

### 1.3 盘前风控预警 (Premarket Risk)
- **大盘情绪温度计**：在每日实际开盘前（07:55），独立调度风控任务（`scripts/premarket_risk_job.py`）。
- **监控标的**：富时 A50 期指（反映 A 股核心资产开盘意愿）与 VIX 恐慌指数（多级降级保障，反映全球高风险避险情绪）。
- **四档盘前风险**：结合 A50 与 VIX，将隔夜外部冲击分为 `NORMAL / CAUTION / RISK_OFF / BLACK_SWAN` 四档。其中 `CAUTION` 表示情绪扰动已经出现，但尚未达到黑天鹅级别；这时允许更轻仓位的试探，不支持激进重仓。
- **物理熔断前哨**：当盘前风险进入 `RISK_OFF` 或 `BLACK_SWAN` 时，系统会优先向“先防守、后进攻”的方向收缩，从宏观面尽量避免在情绪崩溃日盲目开仓。
- **Web 市场信号栏**：登录后页面顶部会统一显示“最新交易日市场信号栏”，同时给出大盘水温、A50（盘前风向标）和 VIX（恐慌指数）三项提示，并用中文文案给出当天的总体操作节奏。

## 2. Step2：Wyckoff Funnel（四层多维漏斗）

### 2.1 Step0：大盘宏观总闸
- **宏观水温：** 通过指数收盘价与 MA50/MA200 交叉、均线短期斜率及单边涨跌幅，归类市场环境（内部标签为 `RISK_ON / NEUTRAL / RISK_OFF / CRASH`，Web 端对用户展示为“偏强 / 中性 / 偏弱 / 极弱”等中文口径）。
- **🌟 强力悬崖检测 (Crash 极寒模式)：** 同步监测主板与小盘股单日暴跌幅度（如 -1.5% 和 -3%），以及市场广度单日断崖式下降（赚了指数不赚钱的虚假诱多）。一旦触发，系统直接切入 `CRASH` 冰封状态。
- **动态自适应调参：** 冷水温时全局提升漏斗参数；遇到 `CRASH` 时漏斗门槛提升至极限（过滤掉 99% 的跟跌股）；热水温适度放开强度过滤区间。

### 2.2 漏斗阶梯与量化因子
#### Layer1：流动性剔除
- 过滤非主板/非创业板样本。
- 采用移动平均平滑，刚性切除流动性干涸、微盘、ST 风险股（**市值 ≥ 35亿**，**近20日均成交额 ≥ 5000万**）。

#### Layer2：全域六通道甄选（精准位阶隔离与独立风控保护）
彻底抛弃单一形态的强行耦合，将股票根据本身位阶划入六大独立信道，互不干扰：
- **主升通道 (Markup)：** 结构必须具备清晰的多头排列（**收盘价及MA50 > MA200**），且挂钩极强的截面相对强度（**要求 RPS50 ≥ 75, RPS120 ≥ 70，且近10日 RPS 斜率 ≥ 0.5**）。同时新增 **MA200 乖离上限 25%**，主动过滤已经远离年线的高位鱼尾股，避免把末端加速错当成健康主升。
- **点火破局 (SOS Bypass)：** 专门捕捉威科夫底部的暴发点火。条件达成（**单日放量突破近60日95%分位数上限且相比近20日均量放大 ≥ 2倍，单日涨幅 ≥ 4.5% 并突破近20日高点**）时，赐予免死金牌，直接无视前序低迷技术指标送入前排。同时要求 **距 MA200 的乖离不超过 20%**，尽量把点火锁定在更低、更干净的位置。
- **潜伏通道 (Ambush)：** 长强短弱（**RPS120 ≥ 70 但 RPS50 ≤ 45**），回调至年线附近（**距离 MA200 的偏离度绝对值 ≤ 8%**）且近期休整（**近20日收益率 ≤ -3%**），专做主升脱离后的首次回首。
- **吸筹通道 (Accumulation)：** 距年内绝版低点极近（**现价 ≤ 年内低点 +35%**），振幅缩窄（**近60日最高最低振幅 ≤ 30%**）、均线极度胶着（**|MA50 - MA200| / MA200 < 6%**）且量能萎缩（**近20日均量 / 前120日均量 < 0.65**），不看短线截面动量，过滤虚假突破。
- **地量蓄势 (Dry Volume)：** 同样具备位置安全垫（**现价 ≤ 年内低点 +35%**），并且侦测到近10日内创造了极小地量（**单日量能 ≤ 近60日成交量的 5% 分位数**），说明卖压已完全被主力耗尽。
- **暗中护盘 (RS Divergence)：** 侦测大盘近期陷入更深冰点（**近20日创下更低点且成交量放大放量超 1.2倍**）的恐慌期时，该股竟然拒绝创新低（**个股近20日低点未破前低，且成交量萎缩至 0.8倍之下**，**现价 ≤ 年内低点 +50%**），为主力隐蔽托底的铁证。

#### Layer3：板块共振过滤与 Top15 Alpha Score 打分
- **共振识别：** 不僵化套用固定前 N 个行业，而是根据”行业通过率（**L2占比 L1 大于 70% 分位**）+ 行业强度截面中位数”构建二重动态过滤。如果小板块强度极高（**强度分位 > 90%**），则直接免死放宽数量门槛。
- **三级放行过滤：** L3 不再是直通层，而是执行实际的板块过滤——核心热门板块直通；次优板块需个股强度 ≥ 70%；极强个股（强度 ≥ 90%）无视板块限制通配放行。安全兜底：若过滤后不足 3 只则全量保留。
- **🌟 兜底排名打分机制（Rank L3）：** 如果在 L4 未捕捉到极度完美的微观形态，则对整体 L3 幸存者做复合因子打分（动量+干涸度+板块加成），强制推送全市场最强共振且技术到位的 Top15 股票（这有效对抗了极度严苛正则导致的空池问题）。

#### Layer4：微观形态捕捉（威科夫扳机）
基于实战演进（容许微小刺穿、真实滑点），对个股执行日线切片捕捉：
- **SOS（点火/跳跃小溪）：** 右侧发令枪，爆量大阳突破界线（**同 L2 点火特征，涨幅 ≥ 4.5%，放量 ≥ 2 倍且突破95%分位数**）。
- **Spring（终极震仓）：** 下探探底蓄势箱体（**近60日支撑，且区间振幅 < 30%**）并强力收回（**放量收回，当日收盘成交量较下探日放大 ≥ 1.15倍**）。
- **LPS（缩量回踩）：** 特定均线支撑位的极值缩量表现（**近3日回踩MA20，且近期均量缩减至前60日均量的 48% 以下**），只保留真正”供应干涸”的最后回踩。
- **EVR (Effort vs Result)：** 高位派发保护下，捕获”放量而不跌”的核心换手背离信号（**近3日量能放大 ≥ 1.3 倍，且换手率 ≥ 1%，股价跌幅 ≤ -2%**）。

## 3. Step3：AI "Tape Reader" (盘口翻译大模型)

抛弃了传统的“把生数据全塞给 AI 试图让其算数”思维，完全依靠 Step2 切片数据驱动 LLM。

### 3.1 高光特征注入（Feature Slice）
- 系统并不把涵盖一切的 OHLC 文本扔给 AI，而是压缩给出**结构背景（均价乖离率验证位阶）**和 **近15日量价切片（缩量确认日）**。
- **核心足迹捕捉：** 专门提炼出过去 60 天内极度缩容、暴涨暴跌或巨量分歧的高光切片日期，供 AI 像作弊一样直接推演主力行为。

### 3.2 双轨制决策与三阵营推演
- 剥离人工设定的复杂压缩策略。
- **双轨制配额（Trend vs Accum）：** 将输入队列拆分为 Trend（主升/点火）与 Accum（潜伏/吸筹/地量/护盘）两条平行轨道，并按大盘水温动态分配容量。当前默认配额 Trend 优先：`RISK_ON=7/5`、`NEUTRAL=5/5`、`RISK_OFF=2/3`（Trend/Accum），回测验证趋势轨道夏普远高于吸筹轨道。
- **硬约束 Prompt：** 强制 AI 在这两条轨道上，以威科夫 Composite Man 的视角，对候选做“逻辑破产 / 储备营地 / 处于起跳板”三阵营审判，而不是把所有候选都硬塞成可交易结论。
- **强制给锚：** 买卖口径不得自创孤立数字，必须依据传入的“最新实际收盘价”框出战区、给出量价确认条件及结构止损。

### 3.3 RAG 舆情防雷
- 结合外部检索引擎实施最后一公里负面信息扫清，一票否决违规、立案等高风险舆情标的。

## 4. Step4：私人再平衡 OMS (订单管理系统)

OMS 层是保证这套系统不出致命灾难的“最终刹车片”。

补充说明：
- Web 端的后台漏斗与后台批量研报，当前使用 GitHub Actions artifact 作为轻量结果承载，不额外引入新数据库表。
- 这样可以在没有独立服务器的前提下，把高内存链路从 Streamlit Community Cloud 挪走，同时保留页面内的交互入口。

### 4.1 独立决策权与系统性物理熔断
- **边界划分：** LLM 输出的文字一律视为“建议”，真正挂单的执行权被全权接管在 Python OMS 里。
- **🌟 极寒熔断阻断器：** 当宏观检测判定为 `CRASH` 模式，或盘前外部风险进入 `RISK_OFF / BLACK_SWAN` 时，OMS 会显著收缩买入权限；在 `CAUTION` 档位下仅保留更轻、更克制的试探空间，不支持激进进攻。
- **优先级防线：** `EXIT（强制止损） > TRIM（止盈） > HOLD > PROBE / ATTACK（建仓）`。

### 4.2 资产与交易纪律
- 依靠 Supabase / 云端环境记录持仓账本。
- 根据总账户权益（NAV）分配硬性预算占比和单次最大资金风险上限。
- 内置 A 股按 100 股取整的刚性要求。剩余资金跑不满 100 股将直接拒单抛出 `NO_TRADE`。

### 4.3 兜底与双轨制退场（Exit 策略）
- **真实账户无脑继承：** 无论是 AI 给出的建议，还是均线跟踪测算，OMS 针对已有持仓将**强制无脑继承用户的实盘止损线**。任何新的风控水位**只能收紧（向上移动止损以锁住利润），绝不允许倒挂放宽**，避免 AI 误判导致实盘亏损扩大。
- **动态 ATR 跟踪防线：** 利用真实波动率的收缩扩张（ATR），对当前所有持仓构建跟随动态平仓线，一旦收盘击穿强制斩仓。
- **通道双轨制退出（抛弃一刀切）：**
  - **吸筹股（底部阶段）：** 锚定建仓底部箱体下沿作为护城河初始止损（**硬止损界线为建仓底部的 -7%**）。利润激活（**上涨 15% 开启移动止损**）。
  - **主升股（高位阶段）：** 抛弃沉没水下的历史最低价，采用**回撤极致止损（近期高点回撤 -10%）**和 **MA50短期防守**相结合；一旦出现极度偏离均值后的高位缩量盘跌（**相对 MA200 偏离度 > 30% 且高位缩量跌破 50% 并连续确认3天**），直接触发 `⚠Distribution（派发）` 警告强制平仓。
- **开盘滑点溢价保护：** 结合静态滑点与 ATR 值算出可接受的买入滑点容忍带，有效避开早盘集合竞价骗炮。
- **买入硬止损：** 对 `PROBE/ATTACK` 开仓动作，OMS 兜底硬止损容忍上限（默认 `-7%`），超出此范围直接拦截。

## 5. 核心流与哲学

1. **先选容错环境，再选股票：** 用大盘广度和均线把风险期一刀切屏蔽。
2. **顺大势，逆小势：** L2 保证标的绝对处于主升浪大区间，L4 则专找大区间内缩量回档并未破结构的小级别买点。
3. **接受实战略有误差，但牢抓高确定性：** L4 放开了一点微观回测容差，配合大模型高光切片的语义辅助，极大提升系统的“人味儿”。
4. **防御体系的终极底线：** 宁可错过，绝不硬抗。基于大盘广度悬崖和 OMS 级订单冻结的双重机制，把躲避非理性黑天鹅（Bull Trap / 系统性崩盘）做成了被动技能。
5. **离线高频引擎：** Dataset Snapshot 打通了本地参数魔调，这才是从脚踏车跃进到工业风洞级别的核心。

量化设计的至高追求，不是从 5000 只股票里精确点出涨停板；而是**“由系统机器代劳，冷冰冰地隔绝 4995 只高危渣滓；把幸存的 5 份附带详尽防守带的盘口档案，呈现于你的桌面之上”**。
